## 設定作品數量

![](./images/2019-07-30-10-57-42.png)

某些頁面的下載以作品為單位，所以會顯示這個設定。它可以設定你要**下載多少個作品**。

>預設值的 -1 表示不限制擷取數量

> 一個作品裡可能有多張圖片

?>設定項右側的灰色文字，提醒你可以輸入的數字的範圍。在不同的頁面裡，這些提示可能不同。

## 設定頁面數量

![](./images/2019-07-30-11-19-03.png)

當你處於**列表頁**時，會看到這個設定。你可以設定你要**下載多少頁**。

>預設值的 -1 表示不限制擷取數量

?>列表頁的下載是向下擷取的，例如你處於第 10 頁，就會從第 10 頁開始下載，而不是從第一頁開始下載。

## 下載作品類型

作品類型選項用來設定你想要下載的作品類型。未被勾選的作品類型不會被下載。

![](./images/20200512174931.png)

?>這些類型是畫師為作品選擇的分類，分為插畫、漫畫、動圖、小說。

```
下載作品型別  ✓全年齡  ✓R-18  ✓R-18G
```

你可以根據分級限制篩選作品。

>在只想下載限制級作品的場景，這個功能比透過設定 “必須的 tag” 為 “R-18,R-18G” 更好用，因為透過 tag 來過濾限制級作品是不夠準確的：某些限制級作品並沒有新增對應的 tag。

這裡可以過濾作品的收藏狀態。你可以選擇下載你收藏的作品，或者未收藏的作品。

![](./images/20200330141038.png)

![](./images/20200401103502.png)

設定下載彩色圖片和（或）黑白圖片 。

如果你設定了過濾彩色或黑白圖片，過濾器會檢查圖片的平均顏色，判斷這個圖片是彩色的還是黑白的。

下載器在擷取時和下載時都會進行此項檢查。

?>有些圖片雖然大部分是黑白的，但還是帶有一些彩色，這樣的圖片是彩色圖片，不屬於黑白圖片。

?>如果你過濾了某種顏色的圖片，那麼在下載過程中，你可能會看到頁面頂部有提示：因為顏色設定，沒有儲存某個檔案。這是因為一個作品可能有多張圖片，並且第一張圖片的顏色可能和其後的圖片的顏色不同。在擷取時，下載器只會檢查第一張圖片，檢查透過的話，就會下載這個作品的所有圖片。之後在下載檔案時，下載器會對後面的所有圖片進行檢查，如果發現後面的圖片不符合要求，就不會下載它。

## 顯示進階設定

![](./images/2021-09-09_233353.png)

控制下載器面板裡是否顯示高階設定。預設為關閉狀態，也就是不會顯示高階設定。

這樣會讓介面更加簡潔，避免因為選項過多導致困擾。

如果開啟這個設定，就會像以前一樣顯示所有的設定。

**提示：** 被隱藏的設定只是在視覺上進行了隱藏，它們仍然會繼續工作。

## 多圖作品只下載前幾張圖片

![](./images/2021-09-09_235112.png)

你可以只下載多圖作品的前幾張圖片。

有些時候，多圖作品的第一張圖片是最有價值的。例如第一張圖是彩色的，後面是未上色的情況。這時候就可以設定你想要的數字，只下載前面幾張。

如果你想避免下載到太多的圖片，也可以使用此設定。

?>如果設定的數字大於作品的圖片數量，將會全部下載。等同於沒有限制數量。

預設未啟用。

## 設定收藏數量

![](./images/2020-12-24_112422.png)

收藏數量，指的是有多少個人收藏了這個作品。收藏的人數越多，說明作品的價值越高。

這個設定有兩種選項：

1. 收藏數量
2. 日均收藏數量

只要作品滿足這兩種選項中的**任意一種**，就會被下載。

預設只啟用了收藏數量。你可以手動啟用“日均收藏數量”。

### 收藏數量

你可以只設置最小值、或者只設置最大值、或者同時設定兩者。

如果你不需要設定某個值，將它留空即可。

?>判斷條件裡包含等於。例如設定最小收藏數量為 500，那麼作品達到 500 收藏就透過，而不是 501。

### 日均收藏數量

日均收藏數量是 `收藏數 / 作品釋出天數` 得到的，設計目的是最佳化對新發布的作品的抓取，尤其是釋出時間不到一天的作品。

例如我們在抓取時設定了收藏數量需要大於 `1000`。有一些近期投稿的作品，雖然質量高，但因為釋出時間短，收藏數量達不到 `1000`，就可以考慮使用這個設定來抓取。

例如一個作品釋出了 6 個小時，有 500 收藏，日均收藏數量會被估算為 2000。如果你設定的日均收藏數量在 `2000` 左右，就可以抓取到它。

提示：

- 下載器不會對發表時間小於 4 個小時的作品檢查“日均收藏數量”，這是為了提高抓取結果的準確性。
- “日均收藏數量”是一個可選的設定，它應該作為補充的設定，而不是唯一的設定。這是因為**日均收藏數量對釋出時間比較長的作品不公平**。
- 你可以透過某些方法單獨使用“日均收藏數量”。首先把最小值、最大值設定為一個不可能的數字 `999999 - 9999999`，再設定“日均收藏數量”，這樣實際上就是單獨使用“日均收藏數量”。

---------

在設定收藏數量時，你必須設定最小值和最大值。

作品的收藏數量增長是一個曲線。一般來說作品剛釋出之後的一段時間內每日收藏數量增長較快；釋出一段時間之後，每日收藏數量會降低。

設想如下情況：

- 作品 A 釋出 1 天，收藏數量為 30。日均 30。
- 作品 B 釋出 5 年，收藏數量 50000。日均 27。

在比較日均收藏數量時，高質量的作品 B 低於普通質量的作品 A。所以這是不合適的。

除非你清除自己要做什麼，否則不要單獨使用“日均收藏數量”。

## 設定寬高條件

![](./images/20200918174448.png)

圖片具有寬度（width）和高度（height）。這個選項設定你想要下載的作品的寬高條件，不符合此條件的作品不會下載。

**提示：**

- 要求圖片的寬高 `>=` 或者 `=` 或者 `<=` 你設定的值。
- 第一個輸入框設定寬度，第二個輸入框設定高度。
- 選擇 “and”，表示圖片的寬高要同時符合設定的寬高。
- 選擇 “or”，表示圖片的寬高要只需要符合寬高設定中的任意一個。
- 預設值都是 0，表示不限制。

?>例如，我們可以設定 `>= 1920 and 1080` 來篩選適合做電腦桌布的圖片。

## 設定寬高比例

![](./images/2020-12-30_123430.png)

你可以設定下載指定形狀的作品：

橫圖、豎圖、正方形。

橫圖的寬度大於高度。豎圖的寬度小於高度。

你也可以設定寬高比。寬高比是寬度除以高度得到的數字。

**提示：**

- 寬高比小於 1 時，圖片是豎圖。寬高比大於 1 時，圖片是橫圖。寬高比越大，圖片越顯得扁長。

## 必須含有 tag

![](./images/2020-11-22_122246.png)

你可以設定下載的作品裡必須包含某些 tag。沒有這些 tag 的作品不會被下載。預設為空。

### 全部

如果你輸入了多個 tag，“全部”選項需要求每個作品都必須同時含有這些 tag。

### 任一

如果你輸入了多個 tag，“任一”選項表示要求每個作品只要含有其中一個 tag 即可。

**提醒：**

- 你可以新增多個 tag，中間用**英文逗號** `,` 分割。
- **不區分大小寫**。
- tag 是嚴格的**全字匹配**。例如你設定了“東方Project”，而某個作品的 tag 裡只有“東方”，那麼它是不符合條件的。反過來也是如此。
- 如果設定了多個 tag，下載器會根據“全部”或者“任一”設定來判斷作品是否符合要求。
- 推薦使用日文（原本的）tag。不推薦使用翻譯後的 tag。

--------

舉個例子，`東方 Project` 的作品有時候標籤不一致，常見的可能有以下三種：

```東方Project,東方,東方プロジェクト```

有的作品可能只有其中一個標籤，所以只搜尋一個 tag 是不完整的。你可以把這三個 tag 全部填入，然後設定模式為“任一”，這樣就不容易漏掉`東方 Project` 的作品了。

## 不能含有 tag

![](./images/2021-09-09_234722.png)

你可以設定下載的作品裡不能包含某些 tag。如果作品有某個 tag 符合此條件，那麼它不會被下載。預設未啟用。

**提醒：**

- 你可以新增多個 tag，中間用**英文逗號** `,` 分割。
- 如果設定了多個 tag，那麼作品**只要符合其中任意一個**，就不會被下載。
- **不區分大小寫**。
- 不能含有 tag 的優先順序比 必須含有 tag 高。如果一個作品符合“必須含有 tag”的條件，也符合“不能含有 tag”的條件，那麼它不會被下載。
- 推薦使用日文 tag。不推薦使用翻譯後的 tag。

### 匹配模式

匹配模式分為“部分一致”和“完全一致”。預設使用“完全一致”。

“完全一致”會使用作品的 tag 與使用者設定的 tag 進行對比，要求兩個 tag 完全相同。

“部分一致”只需要作品的 tag 裡包含使用者設定的 tag 即可，不需要完全相等。

舉個例子：抓取結果中可能有一些腐向（BL）作品，如果你想排除它們，會有一些困難。因為腐向作品的 tag 有很多，而且還在不斷增加中。

如果使用“完全一致”，你必須設定完整的 tag，例如：

```
BL,腐向け,腐向けHQ,Free!,腐ree!,BL松,刀剣亂舞,刀剣亂腐,黒子のバスケ,鬼燈の腐向け,創作BL,◆A【腐】,腐術廻戦,dcst腐向け
```

如果使用“部分一致”，你只需要設定一些出現頻率高的關鍵字，就能排除大部分腐向作品。如：

```
BL,腐
```

**注意：**

“部分一致”不是完全準確的，在某些情況下，會發生錯誤的排除（某些不該排除的作品被排除了）。例如設定 `BL` 可能會導致含有 `blue` tag 的作品被排除。

如果你很在意準確度，請使用“完全一致”模式。

## 命名規則

![](./images/20200907114032.png)

這是很重要的功能，你可以在這裡設定檔案的**命名規則**，也可以**建立資料夾**。

### 標記說明

每一個標記都會在下載時進行替換，比如 `{id}` 會被替換成類似於 `75863159_p0` 的結果。

?>這項設定的右側有一個幫助按鈕（問號 `?`)，可以檢視標記的含義。

- `{id}` 作品的 id，附帶序號，如 44920385_p0
- `{user}` 畫師名字
- `{user_id}` 畫師 id
- `{title}` 作品標題
- `{page_title}` 當前頁面的標題
- `{tags}` 作品的 tag 列表
- `{tags_translate}` 作品的 tag 列表，附帶翻譯後的 tag（如果有）
- `tags_transl_only` 翻譯後的 tag 列表
- `{page_tag}` 當前頁面的 tag。當前頁面沒有 tag 時不可用。
- `{type}` 作品類型，分為 illustration、manga、ugoira
- `{like}` Like count，作品的點贊數。
- `{bmk}` bookmark-count，作品的收藏數。把它放在最前面可以讓檔案按收藏數排序。
- `{view}` View count，作品的瀏覽量。
- `{rank}` 作品在排行榜中的排名。如 #1、#2 …… 只能在排行榜頁面使用。
- `{date}` 作品的建立日期，格式為 yyyy-MM-dd。如 2019-08-29
- `{task_date}` 這次任務擷取完成時的時間。如 2020-10-21
- `{px}` 寬度和高度
- `{series_title}` 系列標題（可能為空）
- `{series_order}` 作品在系列中的序號，如 #1 #2
- `{id_num}` 數字 id，如 44920385
- `{p_num}` 圖片在作品內的序號，如 0、1、2 …… 每個作品都會重新計數。

**注意：**

- 下載時如果檔案已存在，新檔案會覆蓋舊檔案，而不是在後面新增序號。
- 你可以使用多個標記；多個標記之間建議新增分割符號，如 `{id}-{tags}-{user}`，免得各個標記的內容挨在一起，難以分辨。
- 在某些情況下，會有一些標記不可用。不可用的標記不會出現在檔名裡。
- 如果生成的檔名裡有一些特殊字元，不能在下載時使用，會被替換成近似的字元。
- 如果檔名超長，Chrome 會自動截斷超出的部分。
- 除了預設的標記，你也可以自行輸入文字。
- 如果你使用了 `{tags_translate}`，就沒有必要使用 `{tags}`，因為前者包含它。翻譯的內容根據你在 pixiv 的語言設定有所不同。例如，如果你的 pixiv 介面是中文的，那麼 tag 的翻譯一般也是中文的。你可以在首頁底部修改你要是用的語言。
- `tags_transl_only` 只儲存翻譯後的 tag，不儲存原本的日文 tag。但是如果某個 tag 沒有翻譯，則會儲存原本的 tag。
- 檔名裡必須要含有 id（`{id_num}`） 和序號（`{p_num}`）部分，或者直接使用 `{id}`。不然多圖作品裡的圖片的檔名相同，會導致下載時互相覆蓋。
- 沒有後綴名標記，因為字尾名是程式自動新增到檔案末尾的。

### 建立資料夾

**使用斜線 `/` 可以建立資料夾**。例如 `{page_title}/{id}-{tags}-{user}` 會把 `{page_title}` 作為資料夾的名字，後續內容作為檔名。就像這樣：

![](./images/2019-07-30-14-28-59.png)

如果有需要，你也可以使用多個斜線，這樣可以建立多層資料夾。

### 檔案名稱長度限制

![](./images/20200907112513.png)

設定檔名的長度上限。這個長度不包括資料夾部分，但包括字尾名部分。

**一般不需要開啟。**除非當你確定是因為檔名太長導致了問題，才需要啟用這個設定。

一般來說，如果檔名太長的話，Chrome 會自動截斷超出的部分，不會導致儲存失敗。

但是在一些特殊情況下，Chrome 可能不會自動截斷檔名，有可能導致儲存失敗。例如：你把下載位置設定到 NAS 或雲端硬碟時。

## 在不同的頁面型別中使用不同的命名規則

![](./images/2021-09-09_233310.png)

如果你想要在某些頁面裡使用與其他頁面不同的命名規則，可以使用這個設定。

應用場景示例：

- 在使用者的作品列表頁面設定為 `{user}/{id}`，使用使用者名稱建立資料夾。
- 在搜尋 tag 的頁面設定為 `{page_tag}/{id}`，使用這個頁面的 tag 建立資料夾。
- 在排行榜頁面設定為 `{page_title}/{rank}-{id}`，在檔名的前面新增它在排行榜中的編號。

**注意：**
- 啟用這個設定之後，在各種頁面裡的命名規則都會變成下載器預設的規則。你可以把它們修改成你需要的規則。
- 啟用這個設定之後，如果你在下載時切換了頁面型別，可能檔名會發生變化。

例如你在畫師頁面抓取了很多作品，此時的命名規則時 `user/id`。在下載時你進入了作品頁面，此時命名規則變成了 `page_title/id`，那麼下載目錄裡就會出現 `user` 和 `page_title` 兩個資料夾。

## 加入命名標記前綴

![](./images/20200316110148.png)

同時使用多個標記時，檔名裡儲存了多種資訊，如果它們之間沒有標記，容易分不清楚。所以本程式預設會在一些標記前面加上說明。情況如下：

- {title} 生成結果時，前面加上了 title_ 字元。
- {tags} 和 {tags_translate} 生成結果時，前面加上了 tags_ 字元。
- {user} 生成結果時，前面加上了 user_ 字元。
- {userid} 生成結果時，前面加上了 uid_ 字元。
- {bmk} 生成結果時，前面加上了 bmk_ 字元。

其他標記沒有新增說明字元。如果你不想要這些說明字元，可以關閉此選項。

## 第一張圖片不包含序號

![](./images/20200330141245.png)

如果你使用 `{id}` 命名，每個作品的 id 後面都會帶序號，如：

```
80447108_p0
80447108_p1
80447108_p2
```

這個選項會去掉第一張圖的序號部分，變成這樣：

```
80447108
80447108_p1
80447108_p2
```

如果你使用了 `{p_num}` 這個命名標記，那麼第一張圖的序號也會被去掉。

## 在序號前面填充 0

![](./images/2021-09-09_221636.png)

插畫和漫畫作品因為可能有多張圖片，所以是有序號的。

下載器預設不會在序號前新增 0，但是一些軟體對於檔名排序不正確，在這種情況下需要新增 0。

開啟這個設定後，序號的形式可能如下：

```
001
002
003
...
010
011
012
...
```

你可以在子選項裡設定填充後的長度，預設為 3。（因為插畫和漫畫最多可以有 200 p，所以長度應該設定為 3）。

為了便於對序號進行處理，在檔案資料裡添加了 index 欄位儲存其原編號。舊版本沒有 index 欄位，所以對使用 index 的地方做了相容性處理。

## 根據作品型別自動建立資料夾

```
根據作品型別自動建立資料夾  ✓插畫  ✓漫畫  ✓動圖  ✓小說
```

你可以勾選其中**任意多個**選項，這樣下載器會根據作品型別自動建立資料夾。

- 如果勾選了*插畫*，插畫作品會被存入 Illustration 資料夾
- 如果勾選了*漫畫*，漫畫作品會被存入 Manga 資料夾
- 如果勾選了*動圖*，動圖作品會被存入 Ugoira 資料夾
- 如果勾選了*小說*，小說作品會被存入 Novel 資料夾


這個設定可以代替命名標記 `{type}`。`{type}` 總是會為所有型別的作品建立資料夾。使用這個設定，你可以自由的選擇作品型別，你也可以選擇所有作品型別。

### 使用第一個匹配的 tag 建立資料夾 

啟用這個設定之後，使用者可以輸入 tag 列表。

在下載時，如果作品的 tag 列表裡含有使用者設定的 tag，就會使用這個 tag 建立資料夾（僅限第一個匹配到的 tag）。

## 為每個作品建立單獨的資料夾

![](./images/2021-09-09_232308.png)

下載器可以為每個作品建立單獨的目錄，把這個作品裡的檔案儲存進去。

### 檔案數量大於

當作品裡的圖片數量大於設定值的時候，才會為這個作品建立單獨的資料夾。

預設值為 1，也就是隻會為多圖作品建立資料夾。

如果設定為 0，那麼對於單圖作品也會建立資料夾。

### 命名規則

你可以設定此功能建立的資料夾的名稱。可以使用命名規則裡的標記。

**注意：** 

- 在此處不應該使用 `{id}`，請使用 `{id_num}` 代替它。
- 可以使用斜線 `/` 來建立子資料夾。

-----------

以作品 [79239641](https://www.pixiv.net/artworks/79239641 ':target=_blank') 為例，它有 3 張圖片。若命名規則為 `{id}`，在未開啟此功能時，檔名結果如下：

```
79239641_p0.jpg
79239641_p1.jpg
79239641_p2.jpg
```

開啟此功能後，預設會把圖片放進以作品的 id 命名的目錄裡，如下：

```
79239641/
  |---- 79239641_p0.jpg
  |---- 79239641_p1.jpg
  |---- 79239641_p2.jpg
```

### 檔案數量大於

如果從這個作品裡下載的檔案數量大於設定的值，則會為這個作品建立目錄。

預設值為 `1`，當從一個作品裡下載的檔案數量大於 1 時，就會建立目錄。此時，只有一張圖片的作品不會建立目錄；有多張圖片的作品會建立目錄。

如果設定為 `0`，那麼下載的所有作品都會建立一個目錄。

你可以修改這個設定，例如設定為 `3`，只有圖片數量大於 3 的作品會建立目錄。

?>提示：“檔案數量”指的是下載器從這個作品裡下載的檔案數量。如果一個作品含有 10 個圖片，但使用者設定了多圖作品只下載第 1 張，那麼檔案數量就是 1。

### 資料夾名稱使用

你可以選擇目錄的名字：

1. 使用作品 id
2. 使用命名規則

**注意：**

- 如果你使用命名規則作為目錄名字，那麼命名規則不要使用 `{p_num}`，它會導致每個圖片都單獨建立一個目錄。可以考慮使用 `{id}` 替換 `{p_num}`。

## 設定下載執行緒

![](./images/2019-07-30-14-10-49.png)

下載執行緒是指同時進行幾個下載任務，也就是有幾個下載進度條。

**提醒：**

怎樣檢視下載速度呢？以 Chrome 瀏覽器為例，按 `Shift` + `Esc` 鍵開啟它的任務管理器，然後找到進行下載的頁面，就可以看到它的下載速度。

![](./images/2019-07-30-14-17-10.png)

如果下載速度達到 1 MB/s，一般就不會出現下載問題了。如果速度比較慢，那麼我建議你把下載程序設定的小一些。

## 動圖儲存格式

![](./images/20200907113248.png)

當下載動圖作品時，將其儲存為 WebM 影片，或者 GIF 圖片，或者 APNG 圖片，或者 Zip 壓縮包。

**各種格式的特點：**

-  WebM 影片畫質高，體積小。在手機上使用可能不太方便。
-  GIF 格式是常用的動態圖片格式。但是畫質差，體積大。
-  APNG 格式是**無損**畫質的動圖格式。畫質最佳，但是體積最大。
-  把動圖儲存為 zip 格式，不需要轉換，記憶體佔用也小。但是沒有動畫效果。

**提醒：**

- 把動圖轉換為影片需要一些時間，如果下載動圖時，進度條不動了，這是正常的，此時本程式在把動圖轉換為影片。
-  把動圖轉換為影片會佔用更多的記憶體，如果你預料到你要下載很多動圖，建議把下載執行緒設定的小一些，例如設定為 1。
-  動圖的原始檔的體積越大，轉換時的記憶體佔用也越大。

### 保持標籤頁啟用

!>如果要轉換動圖，請保持該頁面處於啟用狀態，否則轉換速度會變慢。

保持該標籤頁啟用的方法：

1. 不要切換到其他頁面。但是這樣就不能瀏覽其他網頁了；
2. 把這個標籤頁拖出來形成一個單獨的視窗，這樣這個標籤頁始終是啟用的。你可以回到之前的視窗瀏覽其他網頁，也不會影響轉換。

保持標籤頁啟用，只需要該頁面在瀏覽器裡是直接顯示的就行，不需要讓這個頁面一直保持顯示在螢幕上（那樣就做不了其他事情了）。

## 不下載重複檔案

![](./images/20200724112527.png)

如果啟用了“不下載重複檔案”選項，那麼下載器會跳過下載重複的檔案。

### 原理

當成功下載了一個檔案之後，下載器會在 IndexedDB 資料庫裡儲存一條記錄，包含 id 和檔名。例如：

```
{
  "id": "85290993_p0",
  "n": "85290993_p0-小日向ほしみ-原創,女孩子,原創,waitress,過膝襪.jpg"
}
```

在下載一個檔案之前，下載器會查詢記錄，判斷這個檔案是不是重複檔案。

!>下載器只會查詢自己的下載記錄，不會檢查硬碟上的檔案（因為沒有這個許可權）。所以這個方法並不完美。

例如你要下載的某個檔案在硬碟上已存在，但是下載器沒有記錄，就仍然會下載它。

相反的，如果你之前下載過某個檔案，之後在硬碟上把它刪除了，但是在下載器裡有記錄，那麼下載器就不會再次下載它（如果你沒有開啟此選項，則可以再次下載）。

### 嚴格策略

當要下載的這個檔案的 **id 以及檔名**都和某個記錄吻合時，則認為這個檔案是重複檔案。

### 寬鬆策略

只判斷 id，不判斷檔名。如果下載的這個檔案的 **id** 和某個記錄吻合，就認為這個檔案是重複檔案。

### 匯出

點選匯出按鈕，可以匯出所有下載記錄，格式為 json 檔案。

>當你要清除 Cookies 資料，或者重灌瀏覽器，你可以先匯出下載記錄，以便進行恢復。

### 匯入

點選匯入按鈕，你可以選擇匯出的下載記錄檔案，匯入到下載器的下載記錄裡。

### 匯入 id 列表

這是個隱藏功能，可以在頁面上輸入口令 `recordtxt` 使用。

以前，下載器只能匯入自己匯出的下載記錄（json 檔案），不支援匯入其他格式的檔案。

但是有使用者想匯入 txt 格式的 id 列表，如：

```
84334853_p0
84334853_p1
84334853_p2
```

所以我添加了這個隱藏功能。

txt 檔案的內容必須是檔案 ID（並且一定要帶序號），否則即使匯入了，和下載器內部的格式不一致，也是無法生效的。

**注意：** 匯入之後可以用“寬鬆”策略來使用。（“嚴格”策略下無效，因為這樣的匯入只有 id，沒有檔名）。

### 清除下載記錄

清空下載器的所有下載記錄。

即使你把下載到硬碟上的檔案刪除掉，但是下載器裡依然會存在著它的下載記錄。當你再次下載時，下載器會認為這個檔案已經下載過了，導致下載器不會再下載它。

如果你遇到了這種情況，可以清除下載記錄，或者關閉“不下載重複檔案”的選項。

- 普通：點選放大鏡圖示後，在圖片檢視器里加載普通尺寸的圖片（最大 1200px * 1200px）。

預設值是“普通”，因為普通尺寸的圖片體積比較小，載入速度快。

這個設定隻影響在圖片檢視器裡顯示的圖片的尺寸，不會影響下載按鈕所下載的圖片的尺寸。

## 背景圖片

你可以選擇自己喜歡的圖片作為下載器的背景圖片，並且可以調節透明度、對齊方式。

**提示：**

建議使用顏色較暗的圖片作為背景圖片。

## 色彩主題

![](./images/2020-11-22_122904.png)

設定下載器的色彩主題，預設為自動偵測。此時下載器會根據 pixiv 的主題是白色還是黑色，使用與之相同的主題。

你可以透過這個選項單獨設定下載器的主題，使其可以不同於 pixiv 的主題。 

下載器的主題目前有白色和黑色。

## 選項卡切換方式

![](./images/2021-09-09_221700.png)

```
○ 滑鼠經過   ○ 滑鼠點選
```

使用者可以選擇使用哪種方式切換下載器的選項卡。

預設方式是滑鼠經過，這樣更快捷。

一些使用者擔心滑鼠經過的方式會誤操作，此時可以選擇滑鼠點選的方式。

## Language

![](./images/20220801_185736.png)

你可以設定下載器使用的語言。這不會影響 Pixiv 的語言。

一般無需設定，保持預設的“自動偵測”即可。這樣下載器會使用和 Pixiv 頁面一致的語言。

除非你想讓兩者不同，比如 Pixiv 使用日語，下載器使用中文，這時才需要修改。

![](./images/2020-11-22_122909.png)

## 根據 sl 建立資料夾

這個一個隱藏設定。開關這個設定的方法：在頁面上用鍵盤鍵入口令：（任選其一）

- `switchsl`
- `kaiguansl`

根據作品的 sl 資料建立資料夾。這個選項預設是關閉的。

sl 是作品資料裡的一個欄位，可能和“色情指數”有關。

sl 的值及其作用：（推測）

- sl 0 的是新發布的普通作品，可能是未進行測定。在釋出後經過一段時間後，會變成其他數值。
- sl 2 的基本都沒什麼色情元素；
- sl 4 的大部分都有一定的色情元素，比如巨乳、大白腿、衣服裸露程度較高，但一般達不到 R-18 的級別。
- sl 6 的大部分都有強烈的色情元素，有很多是 R-18 作品。即使不是 R-18 作品，也會有很多擦邊球。

根據作品的 sl 資料建立資料夾，可能會產生以下幾種名稱的資料夾：

- sl0
- sl2
- sl4
- sl8

>小說作品沒有 sl 資料所以不會用 sl 建立資料夾；只有插畫、漫畫、動圖可以用 sl 建立資料夾。

## 優先下載動圖

這個一個隱藏設定。開關這個設定的方法：在頁面上用鍵盤鍵入口令：（任選其一）

- `dlugoirafirst`
- `qw111`

這個設定開啟的效果：

當下載器抓取完成後，會把結果中的動圖排列到最前面，這樣可以優先下載動圖。

## 滑鼠經過切換選項卡

這個一個隱藏設定。開關這個設定的方法：在頁面上用鍵盤鍵入口令：

```
mouseenter
```

這個選項預設是開啟的。也就是允許滑鼠經過切換選項卡。

有的使用者想要用**滑鼠點選**切換，不想用滑鼠經過來切換，此時就可以輸入口令關閉這個設定。
