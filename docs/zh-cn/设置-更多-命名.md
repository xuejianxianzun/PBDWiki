下载器可以为多图作品添加一层单独的文件夹。在下载器面板上点击该设置的“帮助”按钮可以查看详细说明。

例如以作品 [79239641](https://www.pixiv.net/artworks/79239641 ':target=_blank') 有 3 张图片，启用此功能可以把图片放进以作品 ID 命名的目录里，如下：

```
79239641/
  |---- 79239641_p0.jpg
  |---- 79239641_p1.jpg
  |---- 79239641_p2.jpg
```

**子选项：**

- `图片数量`：当这个作品里的图片数量大于设置值时，下载器才会为其添加这里设置的文件夹。默认是 1，会对所有多图作品生效。你也可以设置为其他值。
- `文件夹规则`：为多图作品添加的文件夹的名称。你可以像设置命名规则一样使用各种标记和自定义文字，也可以使用 `/` 来建立文件夹。默认值是 `{pid}`，也就是使用作品 ID（不带序号）建立文件夹，把文件存放在里面。

之后你还需要修改“图像作品的命名规则”设置，在你需要的地方插入 `/{multi_image_folder}/` 来添加一层文件夹。示例：`pixiv/{user}-{user_id}/{multi_image_folder}/{id}-{title}`

之所以需要手动插入这个标记，是为了让用户可以自行决定这层文件夹的位置。虽然通常它会添加到文件名前面，但有些用户可能需要把它放到其他位置，例如更上层的文件夹，或者文件名里。

**提示：**

- 如果你要在文件夹名字里使用作品 ID，那么不应该使用 `{id}` 标记，而是应该使用 `{pid}`。因为多图作品里的每张图片的 `{id}` 都不同，如果使用 `{id}` 会导致每个图片都会建立一个文件夹。
- 虽然设置名字写着“添加一层文件夹”，但你其实可以在这里设置多层文件夹。
- `{multi_image_folder}` 本身不会创建文件夹（除非你设置的文件夹规则里已经以斜线 `/` 结尾），所以需要在它后面添加斜线 `/` 来建立文件夹。不过有时这也是个可以利用的特性：如果你不想建立文件夹，而是想在文件名里把它标注为多图作品，就可以利用这个特性。例如我把文件夹规则设置为 `多图`，并把它添加到文件名里：`pixiv/{user}-{user_id}/{id}-{multi_image_folder}`，那么文件名里就会有“多图”标记：`pixiv/はまけん-3779820/144008705_p0-多图.jpg`

## 文件名长度限制

<p class="option" data-no="29" style="display: flex;">

你可以设置完整文件名的字符数量限制。完整的文件名包括：文件夹的名字、路径符号、文件名、扩展名。

默认值是 `210`，如果完整的文件名的字符数量超过设置值，下载器会优先截断文件名，使字符数量小于设置值。如果有必要，也会截断文件夹的名字。

**为什么文件名可能会超长？**

命名规则里的一些标签可能会输出很多文字，可能会导致文件名的长度超过操作系统允许的长度（通常不超过 256 个字符）。

有两种标签容易导致此问题：一种是标题，包括 `{title}`、`{page_title}`，因为某些小说的标题很长。另一种是标签列表，包括 `{tags}`、`{tags_translate}`、`{tags_transl_only}`、`{page_tag}`，因为有些标签的字数很多。

### 下载器的处理机制

对于完整的文件路径如 `A/B/C.jpg`，下载器会计算它的字符总数量，如果超过设置值则倒序处理，即优先截断文件名，其次才会截断文件夹的名字。具体处理如下：
1. 如果文件名 `C` 较长则截断它以减少字符数量，但必定会保留 `{id}` 生成的结果，以避免文件名重复。
2. 如果截断 `C` 之后整体长度依然超长，则截断文件夹 `B`。对于每层文件夹，如果它不超过 25 个字符，则不截断它。如果超过 25 个字符才会截断它。另外，每层文件夹在截断之后保留的字符数量会是 25 的倍数。
3. 如果截断 `B` 之后整体长度依然超长，则截断文件夹 `A`。以此类推。
4. 下载器会在被截断的文件夹或文件名的末尾添加一个省略号 `…` 以提醒用户这里发生了截断。

**补充说明：**
- 每个文字或符号都视为 1 个字符。例如，一个英语字母是 1 个字符，一个汉字也是 1 个字符。
- 你可以修改长度限制。默认值 `210` 可能适用于多数用户，不过有些用户可以使用更多的字符数量，有些用户则可能需要使用更小的值。
- 浏览器的下载目录的路径会影响下载器可用的文件名长度。如果下载目录的全路径长度为 30 个字符，那么下载器只能使用 220 个左右的字符。如果下载目录的路径长度比较短，那么下载器就可以使用更多的字符。
- 当文件名超长时，下载器截断之后保留的字符数量通常会小于设置值，而非等于设置值。
- 如果你启用了这个设置（默认启用），那么下载器会先处理一遍文件名，然后再交给浏览器处理。

-----------

**Chrome 浏览器的处理机制：**

Chrome 会先检查每一层文件夹的名字并创建文件夹。如果有任意一层文件夹超长，Chrome 就不会创建它，并且会显示另存为窗口让用户手动保存文件。

如果文件夹的名字没有超长，则检查文件名是否超长。Chrome 可能会自动截断文件名来保存文件，也可能不会。常见的情况如下：
- 在 Windows 系统上，如果文件保存在本地硬盘上，Chrome 通常会自动截断文件名（此时不会显示另存为窗口）。
- 在 Linux 系统上，Chrome 可能始终不会自动截断文件名（总是会显示另存为窗口）。
- 不管在什么系统上，如果浏览器的下载目录是网络位置（而非本地硬盘），Chrome 可能始终不会自动截断文件名（总是会显示另存为窗口）。

如果浏览器不符合预期的显示了另存为窗口，说明你可能遇到了文件名超长的问题（但也可能是其他原因）。此时你可以暂停下载，然后减小这个设置的值，再开始下载。

**使用场景 1：**防止用户改名。

有些用户可能会经常改名，如果你想使用他原来的名字，可以在这里手动设置他的名字。

常见的情况是用户名后面有 @ 符号，例如：

- Anmi@画集発売中
- 奥馬@skeb募集中
- さしみなす@依頼募集中

- いの字/inoji
- 焔すばる★２日目 東C17a
- 送り萬都 🔞仕事募集中
- しりー＊C99木曜東A21b
- ショーンC99木東ユ40b
- オムレットマト西ぬ31b

你可以在这里为他们设置一个固定的名字。

-----------

**使用场景 2：**为用户设置别名，也就是昵称。

例如一个用户的名字是日语，但我不会输入日语，不方便在本机上搜索。如果我为他设置一个中文别名（或者我能够使用的其他语言），就可以方便的搜索了。

如果一个用户的名字很难记住，你也可以为他设置一个容易记住的别名。

## 日期和时间格式

<p class="option" data-no="31" style="display: flex;">
    <span class="settingNameStyle1" data-xztext="_日期格式">日期和时间<span class="key">格式</span></span>
    <input type="text" name="dateFormat" class="setinput_style1 blue" style="width:250px;" value="YYYY-MM-DD">
    <button type="button" class="gray1 textButton showDateTip" data-xztext="_提示">提示</button>
    </p>

下载器的命名规则里有一些标记会生成日期和时间字符串：
- `{date}`
- ` {upload_date}`
- `{task_date}`

它们的默认格式是 `YYYY-MM-DD`（例如 `2021-04-30`），只包含日期，不包含时间。

如果你想修改它们的格式，可以修改这个设置。

对于时间如 `2021-04-30T06:40:08`，可用的标记以及输出结果如下（**区分大小写**）:


- `YYYY` 2021
- `YY` 21
- `MM` 04
- `MMM` Apr
- `MMMM` April
- `DD` 30
- `hh` 06
- `mm` 40
- `ss` 08

## 合并系列小说时的命名规则

你可以在下载器面板里点击该设置的“提示”按钮查看详细说明，此处不再重复。

**注意：**
- 这个设置只会影响合集文件的名字，不会影响单篇小说的文件名。
- 如果这个合集有图片，那么图片的名字也会使用这个设置，使图片的名字与合集文件的名字保持一致。

例如：如果合集文件的名字是 `abcd.epub`，并且单独保存了封面图片，那么图片的名字可能是 `abcd.png`。
